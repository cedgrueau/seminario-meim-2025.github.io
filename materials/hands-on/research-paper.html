Hands-on - Activity 2

Scientific Paper Writing with Quarto Extensions

Time: 20-25 minutes Objective: Learn how to use Quarto extensions to create professionally formatted scientific papers.

Overview

In this activity, you’ll explore how Quarto extensions can transform your document into a properly formatted scientific paper. We’ll use the IEEE template, one of the most common formats in computer science publications.

Step 1: Install the IEEE Extension (5 minutes)

Objective: Set up a new document using the IEEE template.

Tasks:

  1. Open terminal/console in your IDE or system terminal

  2. Create new directory:

mkdir ieee-paper
cd ieee-paper
  1. Install the extension:
quarto use template dfolio/quarto-ieee
  1. Verify installation: Wait for the success message

Troubleshooting:

  • If you get a “command not found” error, ensure Quarto is properly installed
  • On some systems, you might need to restart your IDE after installation
  • Latex versions conflict:
    • You can manage TinyTeX directly from the terminal. This is the most straightforward method.
      • Check your current installation by running the following command in your terminal to list installed tools and their versions:
      quarto list tools
      • Update TinyTeX with this command:
      quarto update tinytex
      • This command will download and install the latest version of TinyTeX for you.

Step 2: Explore IEEE Paper Project (5 minutes)

Objective: Explore the structure of IEEE template.

Tasks:

  1. Explore generated files:
  • paper.qmd - Main paper document (the name of the file can differ according to the project creation)
  • references.bib - Bibliography file
  • _quarto.yml - Paper metadata

Step 3: Explore the Paper Structure (5 minutes)

Objective: Understand how scientific papers are structured in Quarto.

Tasks:

  1. Open paper.qmd and examine these key sections:

YAML Header - Notice the academic-specific metadata:

title: "The Title of the Paper"
author:
  - name: First Author
    affiliations: 
      - name: University One
        department: Department Name
    email: author1@university.edu
  - name: Second Author
    affiliations: 
      - name: University Two
        department: Department Name
    email: author2@university.edu
abstract: |
  This is the abstract section...
keywords:
  - keyword1
  - keyword2
journal: "Journal Name"
format: ieee

Paper Sections - Identify these academic paper elements:

  • Abstract section
  • Introduction with citation [@citation]
  • Mathematical equations and theorems
  • Figures with captions and cross-references
  • Tables with proper formatting
  • Conclusion and future work
  • Bibliography section

Step 4: Render and Analyze (5 minutes)

Objective: Generate the formatted paper and identify key features.

Tasks:

  1. Render the paper:
quarto render paper.qmd

  1. Open paper.pdf and examine the output

  2. Answer these questions by comparing the source .qmd and output .pdf:

Q1: Citations & Bibliography

  • How are citations formatted in the text?
  • What syntax is used for multiple citations [@cite1; @cite2]?
  • How does the bibliography get automatically generated?

Q2: Mathematical Content

  • How are equations numbered and referenced?
  • What’s the difference between $$...$$ and \[...\]?
  • How are theorem environments created?

Q3: Figures & Tables

  • How are figures captioned and cross-referenced?
  • What makes tables academic-style?
  • How are multiple subfigures handled?

Q4: Academic Structure

  • What special sections does IEEE require?
  • How are author affiliations formatted?
  • Where does the abstract appear?

Step 5: Key Syntax Identification (5 minutes)

Objective: Document the essential Quarto syntax for scientific writing.

Tasks: Based on your exploration, identify how to create:

  1. Citations:
Previous work [@smith2020] shows that...
Multiple studies [@jones2019; @lee2021] confirm...
  1. Mathematical elements:
\begin{equation}
E = mc^2
\end{equation}
  1. Figures with cross-references:
![Caption text](figure.png){#fig-id}

As shown in Figure @fig-id...
  1. Theorems and proofs:
::: {#thm-pythagorean}
**Pythagorean Theorem**: In a right triangle...
:::
  1. Academic metadata:
abstract: |
  Multi-line abstract
  with proper formatting.
keywords: [keyword1, keyword2, keyword3]

Discussion Points

Extension Benefits:

  • What formatting rules does the IEEE extension automatically apply?
  • How does this compare to manually formatting in Word/Latex?
  • What other journal extensions might be useful?

Workflow Advantages:

  • How can version control (Git) help with paper collaboration?
  • What’s the benefit of having content separate from formatting?
  • How does this support reproducible research?

Optional Extension: Customize Your Paper

If you have extra time, try these modifications:

  1. Add your own abstract in the YAML header
  2. Insert a simple figure with cross-reference
  3. Add a citation to the references.bib file and cite it in text
  4. Create a simple table with proper academic formatting
# Bonus: Explore other available extensions
quarto list extensions | grep article

Key Takeaways

  • Extensions provide professional templates with one command
  • YAML metadata handles academic paper requirements
  • Cross-references automatically update in final output
  • Bibliography management is integrated and automated
  • Multiple output formats from the same source
