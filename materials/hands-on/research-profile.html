Hands-on - Activity 1

Multi-Format Publishing: Researcher Profile

Objective: Create a single Quarto document that describes yourself and your research, then render it to multiple formats.

Duration: 15-20 minutes

Prerequisites: Basic Quarto installation with latex (tinytex) and fundamental Markdown knowledge

NoteInstallation Guide

Please complete the Quarto installation to perform this activity.

Starter new project

Create a new project / directory on your PC and open it with your IDE.

Starter Document Template

Create a file called research-profile.qmd with this initial content, replacing the author value with your name:

---
title: "Research Profile"
author: "Your Name"
date: "today"
format:
  html:
    theme: cosmo
    toc: true
  pdf: default
  revealjs: 
    transition: slide
    slide-level: 2
    output-file: "research-profile-slides.html"
---

Step 1: YAML Configuration (5 minutes)

Objective: Configure the document header for multi-format output.

Tasks:

  1. Basic metadata: Fill in your name, and adjust the date
  2. Format configuration: Ensure all three output formats are specified
  3. Add language support (optional): Add lang: en for English
NoneKey Concepts
  • The format section declares all output types
  • Each format can have its own specific options
  • slide-level: 2 tells RevealJS to use level 2 headings (##) as new slides

Step 2: Content Creation (10 minutes)

Objective: Add structured content that works well across all formats.

Add this content below the YAML header, after the --- symbol, replacing the text with information about your you and your work:

## About Me

I am a researcher focused on **data science methodologies** and their application to environmental studies. My work integrates:

- Statistical modeling
- Reproducible research practices  
- Open science principles

## Research Topic

My thesis investigates *climate pattern analysis* using machine learning approaches. The primary research question examines how temporal sequences can improve predictive accuracy.

::: {.callout-note}
## Research Context
This work builds on previous climate modeling approaches but introduces novel temporal sequencing.
:::

## Methodology

The analysis employs recurrent neural networks with LSTM architecture. The basic formula for the hidden state update is:

$$
h_t = \tanh(W_{hh}h_{t-1} + W_{xh}x_t + b_h)
$$

## Project Timeline

```{mermaid}
gantt
    title Research Timeline
    dateFormat YYYY-MM
    section Data Collection
    Literature Review :done, 2023-01, 2023-04
    Field Data Gathering :active, 2023-05, 2024-02
    section Analysis
    Model Development :2023-09, 2024-06
    Validation :2024-03, 2024-08
```

## Acknowledgments

This research is supported by the Environmental Data Science Initiative.

## Contact Information

- **Email**: your.name@university.edu
- **Office**: Computer Science Building, Room 304
- **Website**: [https://your-lab-website.edu](https://your-lab-website.edu)

Content Design Notes: - Headings: Structure with level 2 headings (##) that become slides in RevealJS - Mixed elements: Include bold, italics, lists, math, code, and Mermaid diagrams - Format compatibility: All chosen elements work across HTML, PDF, and RevealJS

Step 3: Basic Styling (5 minutes)

Objective: Apply simple theme customizations.

Tasks:

  1. Experiment with HTML themes: Try changing cosmo to journal, flatly, or darkly
  2. Adjust RevealJS transition: Change slide to fade or zoom
  3. Add custom colors (optional extension):
format:
  html:
    theme: 
      light: cosmo
      dark: darkly
    highlight: tango

üõ†Ô∏è Rendering Instructions

You can render the document using quarto CLI ou using the menu commands.

Quarto CLI

To render all formats:

quarto render research-profile.qmd --to all

To render specific formats:

quarto render research-profile.qmd --to html
quarto render research-profile.qmd --to pdf  
quarto render research-profile.qmd --to revealjs

Positron or VSCode commands

  1. Activate the Command Palette under the view menu or using CTRL+SHIFT+P
  2. Insert quarto to visualize quarto commands and choose Quarto: Render Document or Quarto: Preview.

Troubleshooting Tips

  • Mermaid diagrams: Ensure you have the Mermaid extension installed
  • Math rendering: All formats support LaTeX math, but PDF may require LaTeX installation
  • RevealJS slides: Use ## headings for new slides, ### for sub-sections

Objective: Create tabbed sections for better organization in HTML output.

Tasks:

  1. Wrap content in tabset: Add this structure around your sections:

::: {.panel-tabset}
## Section 1
Content here

## Section 2
More content
:::

  1. Apply to research sections: Try wrapping your ‚ÄúResearch Topic‚Äù and ‚ÄúMethodology‚Äù sections

  2. Test the output: Render to HTML and click between tabs

NoneKey Concepts:
  • Tabsets only work in HTML output (not PDF)
  • Uses CSS classes from Bootstrap
  • Perfect for organizing related content

Extension 2: Add Speaker Notes to RevealJS (5 minutes)

Objective: Include presenter notes that only you see during presentations.

Tasks:

  1. Add notes after any slide:

## Research Topic

My thesis investigates climate pattern analysis...

::: {.notes .hidden}
Remember to mention the collaboration with meteorology department here.
Additional statistics are in the backup slide.
:::
  1. Include multiple note sections: Add notes to 2-3 different slides
  2. Test in presentation: Press S during RevealJS presentation to see speaker view.
NoneKey Concepts:
  • Notes are hidden from audience view
  • Access speaker view with S key
  • Useful for reminders and additional context
  • The .hidden class is a css class that ensure that the note is not visible in the HTML document.

Objective: Include and style an image in your document.

Tasks:

1.Add image file: Place your photo in the project folder as profile.jpg 2. Insert with styling:

NoneKey Concepts:

  • Apply CSS classes with {.class-name}
  • Inline styles with style="property: value"

Extension 4: Custom CSS Styling (10 minutes)

Objective: Apply fine-grained control over document appearance.

Tasks:

  1. Create CSS file: Make styles.css in your project folder
  2. Add custom styles:
/* Custom profile styling */
.profile-img {
  border: 3px solid #2c3e50;
  box-shadow: 0 4px 8px rgba(0,0,0,0.1);
}

/* Color scheme for headers */
h1, h2, h3 {
  color: #2c3e50;
  border-bottom: 1px solid #ecf0f1;
  padding-bottom: 0.3em;
}

/* Custom block styling */
.acknowledgments {
  background-color: #f8f9fa;
  padding: 15px;
  border-left: 4px solid #3498db;
}
  1. Link CSS in YAML:
format:
  html:
    theme: cosmo
    css: styles.css
  1. Apply custom class:

:::{.acknowledgments}

## Acknowledgments

This research is supported by the Environmental Data Science Initiative.
:::
NoneKey Concepts:
  • CSS provides pixel-level control
  • Link external files in YAML header
  • Combine with Quarto classes for powerful layouts
